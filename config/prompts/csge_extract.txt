== 输出要求 ==
请严格输出 JSON，不要输出额外解释，不要生成 schema 之外的关系。
输出格式如下：
{{
  "entities": [
    {{
      "name": "实体名",
      "type": "实体类型英文标识",
      "attributes": {{"属性名": "属性值"}}
    }}
  ],
  "triplets": [
    {{
      "head": "头实体名",
      "head_type": "头实体类型英文标识",
      "relation": "关系英文标识",
      "tail": "尾实体名",
      "tail_type": "尾实体类型英文标识",
      "confidence": 0.90,
      "evidence": "支撑该关系的最短原文证据片段",
      "source_corpus": "descriptive"
    }}
  ]
}}

== 抽取规则 ==
1. 关系名称必须严格使用英文标识，只能来自允许关系列表。
2. 如果文本只支持属性事实，不支持八类诊断关系，不要硬造关系。
3. 至少一端应与当前 POI 或其所属区直接相关，避免抽取泛化知识。
4. `source_corpus` 只能取 `descriptive`、`demand`、`policy` 之一。
5. `evidence` 必须是输入文本里的直接证据片段，不要编造。
6. 仅输出 `confidence >= 0.60` 的 triplet。
7. 不要输出旧版关系，如“属于”“始建于”“邻近”“同区”“同类”“关注于”等。
