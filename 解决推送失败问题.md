# 解决 Git 推送失败问题

## 问题诊断

根据错误信息:
```
error: RPC failed; curl 55 Send failure: Connection was reset
send-pack: unexpected disconnect while reading sideband packet
Writing objects: 100% (290/290), 110.91 MiB | 4.13 MiB/s, done.
fatal: the remote end hung up unexpectedly
```

**问题原因**:
1. 项目太大 (110.91 MiB)
2. 网络连接不稳定,在传输过程中被重置
3. GitHub 对 HTTPS 推送有大小和时间限制

## 解决方案

### 方案 1: 使用 GitHub Desktop (强烈推荐,最可靠)

GitHub Desktop 会自动处理大文件推送和网络重试:

1. **下载安装**: https://desktop.github.com/
2. **登录账号**: qtyt-dsht
3. **添加仓库**: 
   - File → Add local repository
   - 选择: `D:\个人论文\Christy\github_open_source\CulturLand-Check`
4. **推送**: 点击 "Publish repository" 或 "Push origin"

GitHub Desktop 会自动:
- 分块传输大文件
- 网络中断时自动重试
- 显示进度条

### 方案 2: 压缩大文件后推送

某些大文件可能导致推送失败,让我们检查并压缩:

```bash
cd D:\个人论文\Christy\github_open_source\CulturLand-Check

# 查找大文件 (>10MB)
git rev-list --objects --all | git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | awk '/^blob/ {print substr($0,6)}' | sort -n -k 2 | tail -20

# 如果有超大文件,考虑使用 Git LFS
git lfs install
git lfs track "*.shp"
git lfs track "*.dbf"
git add .gitattributes
git commit -m "Add Git LFS tracking"
git push -u origin master
```

### 方案 3: 使用 SSH 协议 (更稳定)

SSH 协议比 HTTPS 更稳定,适合大文件传输:

#### 步骤 1: 生成 SSH 密钥

```bash
# 生成密钥
ssh-keygen -t ed25519 -C "1143326364@qq.com"

# 查看公钥
type %USERPROFILE%\.ssh\id_ed25519.pub
```

#### 步骤 2: 添加到 GitHub

1. 复制公钥内容
2. 访问: https://github.com/settings/keys
3. 点击 "New SSH key"
4. 粘贴公钥并保存

#### 步骤 3: 切换到 SSH 并推送

```bash
cd D:\个人论文\Christy\github_open_source\CulturLand-Check

# 切换到 SSH URL
git remote set-url origin git@github.com:qtyt-dsht/TD-GRAG.git

# 推送
git push -u origin master
```

### 方案 4: 分批推送 (手动控制)

如果网络特别不稳定,可以分批推送:

```bash
cd D:\个人论文\Christy\github_open_source\CulturLand-Check

# 创建一个轻量级的初始提交
git checkout --orphan temp_branch
git add README.md .gitignore requirements.txt
git commit -m "Initial commit: Basic files"
git push -u origin temp_branch:master

# 然后逐步添加更多文件
git add config/
git commit -m "Add config files"
git push origin temp_branch:master

git add src/
git commit -m "Add source code"
git push origin temp_branch:master

# 继续添加其他目录...
```

### 方案 5: 增加网络超时和重试

```bash
cd D:\个人论文\Christy\github_open_source\CulturLand-Check

# 增加超时时间
git config http.postBuffer 524288000
git config http.lowSpeedLimit 0
git config http.lowSpeedTime 999999

# 使用压缩
git config core.compression 9

# 再次尝试推送
git push -u origin master
```

### 方案 6: 使用代理或 VPN

如果是网络环境问题:

```bash
# 如果有代理
git config http.proxy http://127.0.0.1:7890
git config https.proxy http://127.0.0.1:7890

# 推送
git push -u origin master

# 推送后取消代理
git config --unset http.proxy
git config --unset https.proxy
```

### 方案 7: 排除大文件,使用 Git LFS

检查哪些文件最大:

```bash
cd D:\个人论文\Christy\github_open_source\CulturLand-Check

# Windows PowerShell 查找大文件
Get-ChildItem -Recurse | Where-Object {!$_.PSIsContainer} | Sort-Object Length -Descending | Select-Object -First 20 FullName, @{Name="Size(MB)";Expression={[math]::Round($_.Length/1MB,2)}}
```

如果发现超大文件 (>50MB):

```bash
# 安装 Git LFS
git lfs install

# 追踪大文件类型
git lfs track "*.shp"
git lfs track "*.dbf"
git lfs track "*.json"

# 提交 LFS 配置
git add .gitattributes
git commit -m "Configure Git LFS"

# 重新添加大文件
git add .
git commit --amend --no-edit

# 推送
git push -u origin master
```

## 推荐执行顺序

1. **首选**: 使用 GitHub Desktop (最简单可靠)
2. **备选**: 使用 SSH 协议 (更稳定)
3. **最后**: 如果以上都不行,考虑 Git LFS 或分批推送
